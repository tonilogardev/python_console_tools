# Engineering Guidelines (python_console_tools)\n\n## Branching\n- `main`: estable, sólo merges vía PR.\n- `codex/main_dev_pro`: rama de integración continua (features largas). \n- `codex/main_dev_pro_interface`: trabajo actual de interfaz/CLI; hacer PR hacia `codex/main_dev_pro` y luego a `main`.\n- Convención para nuevas ramas: `codex/<area>-<breve>` (ej. `codex/feat-auth`, `codex/chore-ci`).\n\n## Commits y PR\n- Formato commit: `type: mensaje` (conventional commits: feat, fix, chore, docs, refactor, test).\n- PR checklist:\n  - `make lint format type test` pasa.\n  - Sin secretos (API keys, tokens) ni `.env`.\n  - Cobertura mantenida (objetivo ≥85%).\n  - Descripción clara: qué cambia, por qué, riesgos.\n\n## Estilo Python\n- Typing obligatorio: funciones nuevas con anotaciones; `mypy` debe pasar.\n- Lint/format: `ruff format` + `ruff check` (usa reglas en `pyproject.toml`).\n- Evitar `print`; usar logging.\n- Manejo de errores: `try/except` con mensajes útiles y logging, no silenciar excepciones sin razón.\n- IO/paths: usar `pathlib`; parametrizar rutas.\n- CLI: subcomandos con Typer; validaciones de entrada y mensajes de ayuda.\n\n## Logging\n- Config vía `configs/logging.dev.yaml`; no hardcodear niveles.\n- Usa logger por módulo: `logger = logging.getLogger(__name__)`.\n- Mensajes estructurados y accionables; incluye contexto (ids, recurso) pero nunca secretos.\n\n## Configuración y secretos\n- Valores en `.env` (ver `.env.example`); no commitear `.env`.\n- No guardar credenciales en código ni en logs.\n- Preferir variables de entorno para endpoints, tokens, rutas sensibles.\n\n## Dependencias\n- Añadir deps en `pyproject.toml`; `pip install -e .[dev]` mantiene editable.\n- Para nuevas libs, justificar en PR (seguridad/superficie de ataque mínima).\n- Ejecutar `pip-audit` si se añaden deps sensibles.\n\n## Tests\n- Framework: `pytest` (+ `coverage`).\n- Nuevas funciones públicas: añadir tests.\n- Usa fixtures en lugar de mocks ad-hoc cuando aplique.\n- Objetivo: cobertura ≥85% global; evita pruebas frágiles (dependencias de tiempo/red).\n\n## Seguridad\n- Ejecutar `bandit -r src` para cambios con IO, subprocess o crypto.\n- Validar entradas de usuario/CLI; no eval/exec de entrada.\n- No escribir archivos fuera de rutas permitidas; crear directorios con permisos mínimos.\n\n## Release / CI (pendiente)\n- Workflow GH Actions a añadir: matrix 3.11/3.12; pasos `ruff check`, `ruff format --check`, `mypy`, `pytest` con coverage; cache de pip.\n- Protecciones de rama recomendadas: requerir PR + checks verdes para `main` y `codex/main_dev_pro`.\n\n## Definition of Done (por cambio)\n- Código y tests actualizados.\n- Lint/format/type/tests/coverage ejecutados o justificados.\n- Docs actualizadas (README o sección relevante).\n- PR abierto con contexto y checklist marcada.\n*** End Patch
